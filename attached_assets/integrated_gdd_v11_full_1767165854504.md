# 1장. 게임 소개

## 1.1. 게임명

**글로벌 엠파이어: 왕좌의 게임 (Global Empire: The Game of Thrones)**

본 게임은 전 세계를 무대로 한 대규모 멀티플레이어 실시간 전략 시뮬레이션 게임입니다. 플레이어는 하나의 도시를 통치하는 영주가 되어 외교, 경제, 전쟁, 첩보 활동을 통해 자신의 영향력을 확장하고, 최종적으로 세계 패권을 차지하는 것을 목표로 합니다.

## 1.2. 장르 정의

본 게임은 다음과 같은 장르적 특성을 결합합니다:

| 장르 요소 | 설명 |
| :--- | :--- |
| **실시간 턴제 (Real-Time Turn-based)** | 30~60초 단위의 짧은 턴을 통해 실시간으로 빠르게 전개되는 게임 플레이 |
| **4X 전략 (eXplore, eXpand, eXploit, eXterminate)** | 탐험, 확장, 개발, 정복의 4가지 핵심 요소를 모두 포함 |
| **RTS (Real-Time Strategy)** | 실시간으로 병력을 이동하고 전투를 수행하는 전략적 요소 |
| **웹 기반 (Web-based)** | 별도의 설치 없이 웹 브라우저에서 즉시 플레이 가능 |
| **멀티플레이어 (Multiplayer)** | 최대 100명의 플레이어가 하나의 월드에서 동시에 경쟁 |

## 1.3. 게임 철학 및 목표

### 1.3.1. 핵심 철학

본 게임은 플레이어의 **전략적 깊이와 자유도**를 극대화하는 것을 핵심 철학으로 삼습니다. 정형화된 선택지를 넘어, 플레이어가 직접 텍스트로 전략을 수립하고, LLM(거대 언어 모델)이 이를 분석하여 전투에 반영하는 혁신적인 시스템을 통해 예측 불가능하고 창의적인 플레이 경험을 제공하고자 합니다.

### 1.3.2. 설계 원칙

| 원칙 | 설명 |
| :--- | :--- |
| **자유로운 전략 수립** | 정해진 틀을 벗어나 자신만의 창의적인 군사 전략을 텍스트로 수립 |
| **AI 기반 판정** | LLM이 전략의 논리성, 지형 활용도, 병과 연계성을 분석하여 공정한 판정 |
| **PVP 중심 설계** | 플레이어 간의 경쟁과 협력이 게임의 핵심 동력 |
| **실시간 속도감** | 짧은 턴 시간을 통해 지루함 없이 빠르게 진행되는 게임 플레이 |
| **깊이 있는 경제** | 자원 관리, 건물 건설, 거래 시스템을 통한 전략적 경제 운영 |

### 1.3.3. 핵심 경험

플레이어가 본 게임을 통해 경험하게 될 핵심 요소는 다음과 같습니다:

**문명의 깊이**: 도시 성장, 건물 건설, 자원 관리, 기술 발전(추상화) 등 제국을 운영하는 깊이 있는 경험을 제공합니다. 플레이어는 단순히 병력을 모아 공격하는 것이 아니라, 장기적인 관점에서 도시를 발전시키고 경제를 운영해야 합니다.

**RTS의 속도감**: 30~60초 단위의 짧은 턴을 통해 실시간으로 빠르게 전개되는 게임 플레이를 경험합니다. 플레이어는 주어진 시간 내에 신속하게 판단하고 명령을 내려야 하며, 이는 긴장감 넘치는 게임 플레이를 제공합니다.

**텍스트 전략의 자유도**: 정해진 틀을 벗어나 자신만의 창의적인 군사 전략을 텍스트로 수립하고 AI의 판정을 받는 새로운 방식의 전투를 경험합니다.

## 1.4. 승리 조건

### 1.4.1. 정복 승리 (Conquest Victory) - 기본

게임의 최종 승리 조건은 **정복 승리(Conquest Victory)**입니다. 한 명의 플레이어 또는 하나의 동맹이 월드에 존재하는 **100개의 모든 도시를 점령**하는 즉시 게임은 종료되며, 해당 플레이어 또는 동맹이 최종 승리자가 됩니다.

### 1.4.2. 턴 제한 승리 (Turn Limit Victory) - 선택

방 설정에 따라 **300턴 도달 시점의 점수 1위**가 승리하는 턴 제한 승리 옵션을 선택할 수 있습니다. 이는 게임이 지나치게 길어지는 것을 방지하기 위한 옵션입니다.

### 1.4.3. 점수 계산 방식

턴 제한 승리 시 점수는 다음과 같이 계산됩니다:

| 항목 | 점수 |
| :--- | :---: |
| 수도 보유 | +100점/개 |
| 주요 도시 보유 | +50점/개 |
| 일반 도시 보유 | +10점/개 |
| 작은 마을 보유 | -10점/개 (유지 부담) |
| 총 돈 보유량 | 보유량 / 1,000 |
| 총 특산물 가치 | 가치 / 500 |
| 총 병력 | 병력 / 100 |
| 전투 승리 횟수 | 승리 × 5 |
| 동맹 관계 | 동맹 × 10 |
| 성공한 거래 횟수 | 거래 × 2 |

## 1.5. 플레이 타임

### 1.5.1. 예상 플레이 타임

| 게임 모드 | 턴 시간 | 예상 총 플레이 타임 |
| :--- | :---: | :---: |
| **빠른 게임** | 30초/턴 | 약 1.5~2시간 |
| **표준 게임** | 45초/턴 | 약 2~3시간 |
| **긴 게임** | 60초/턴 | 약 3~4시간 |

### 1.5.2. 세션 관리

플레이어는 게임 중간에 나갔다가 다시 돌아올 수 있습니다. 방에 접속한 플레이어가 0명일 경우, 해당 방은 즉시 비활성 상태로 변경되며 턴 진행, 자원 생산, 재화 충전이 모두 중단됩니다. 비활성 상태가 7일(168시간) 지속될 경우, 해당 방의 모든 데이터(DB)는 영구 삭제됩니다.

# 2장. 실시간 턴 시스템 (Time Flow)

## 2.1. 동시 턴 진행 방식

### 2.1.1. 기본 개념

모든 플레이어는 동일한 턴 카운트를 공유하며, 각 턴은 방 설정에 따라 **30초, 45초(기본), 60초** 중 하나로 설정된 시간 동안 진행됩니다. 모든 플레이어는 주어진 시간 내에 자신의 모든 명령을 내리고, 턴이 종료되면 모든 명령이 동시에 처리되는 동시 턴 진행(Simultaneous Turn-based) 방식을 채택하여 대기 시간을 최소화하고 속도감을 높입니다.

### 2.1.2. 턴 동기화

모든 플레이어는 같은 턴 카운트 (T1, T2, T3...)를 공유합니다. 각 플레이어의 액션은 독립적이지만 같은 턴 내에 처리됩니다. 전투 중인 영주는 전투 맵에서 진행 중이고, 외교 중인 영주는 외교 채팅 중이며, 일반 플레이 중인 영주는 지도에서 계속 진행하지만, 모든 영주가 동시에 같은 턴 수를 진행합니다.

## 2.2. 턴 페이즈 상세

### 2.2.1. 턴 시작 (T-Start) - 1초

턴이 시작되면 다음 항목들이 자동으로 처리됩니다:

| 처리 항목 | 상세 내용 |
| :--- | :--- |
| **자원 생산** | 돈, 병력, 식량, 특산물이 도시별로 생산됨 |
| **병력 회복** | 부상병이 병원 건물 효과에 따라 회복됨 |
| **유지비 차감** | 병력 유지비(식량)가 자동 차감됨 |
| **건설 진행** | 건설 중인 건물의 남은 턴 수가 1 감소 |
| **AI 의사결정** | AI 영주의 공격, 거래, 외교 결정이 처리됨 |
| **첩보 활동** | 파견된 스파이의 첩보 활동이 진행됨 |

### 2.2.2. 액션 페이즈 (Action Phase) - 28~58초

플레이어가 명령을 내릴 수 있는 시간입니다. 다음과 같은 명령을 내릴 수 있습니다:

| 명령 유형 | 상세 내용 |
| :--- | :--- |
| **병력 이동** | 병력을 다른 타일로 이동 명령 |
| **건설 시작** | 새로운 건물 건설 명령 |
| **스파이 파견** | 스파이를 적 도시에 파견 |
| **거래 제안** | 다른 플레이어에게 거래 제안 |
| **외교 활동** | 동맹 제안, 선전포고 등 |
| **전투 전략 입력** | 전투 발생 시 전략 텍스트 입력 |
| **병력 징집** | 도시 인구에서 병력 징집 |
| **세율 조정** | 도시의 세율 조정 |

입력된 명령은 서버 큐(Queue)에 쌓이며, 턴 종료 시 일괄 처리됩니다.

### 2.2.3. 턴 종료 및 판정 (Resolution) - 4초

턴이 종료되면 다음 항목들이 처리됩니다:

| 처리 항목 | 상세 내용 |
| :--- | :--- |
| **전투 판정** | LLM이 전투 전략을 분석하고 결과 판정 |
| **이동 실행** | 병력 이동 명령이 실행됨 |
| **거래 체결** | 양측이 수락한 거래가 체결됨 |
| **뉴스 생성** | 주요 이벤트에 대한 뉴스가 생성됨 |
| **상태 업데이트** | 도시 상태, 외교 관계 등이 업데이트됨 |

### 2.2.4. 다음 턴 전환 - 즉시

판정이 완료되면 즉시 다음 턴으로 전환됩니다.

## 2.3. 비동기 처리

### 2.3.1. 전투 중 다른 활동

전투가 발생하더라도 다른 플레이어의 턴 진행은 중단되지 않습니다. 전투 중인 플레이어만 전투 화면에서 전략을 입력하고, 다른 플레이어는 평소처럼 자신의 도시를 관리할 수 있습니다.

### 2.3.2. 외교 중 다른 활동

외교 협상 중에도 다른 활동이 가능합니다. 외교 채팅은 별도의 창에서 진행되며, 플레이어는 동시에 지도를 확인하고 다른 명령을 내릴 수 있습니다.

# 3장. 플레이어 및 세션 관리

## 3.1. 방 생성 및 설정

### 3.1.1. 방 생성 단계

방을 생성하는 플레이어(방장)는 다음 단계를 거칩니다:

| 단계 | 설정 항목 | 옵션 |
| :---: | :--- | :--- |
| 1 | 방 이름 입력 | 자유 입력 (최대 30자) |
| 2 | 게임 모드 선택 | 랭크드 / 캐주얼 / 커스텀 |
| 3 | 플레이어 수 설정 | 2~100명 (AI 포함) |
| 4 | AI 난이도 선택 | 쉬움 / 보통 / 어려움 |
| 5 | 턴 시간 설정 | 30초 / 45초 / 60초 |
| 6 | 맵 선택 | 지구 고정 맵 / 랜덤 모드 |
| 7 | 승리 조건 설정 | 정복 승리 / 턴 제한 승리 |

### 3.1.2. 게임 모드 상세

| 모드 | 설명 |
| :--- | :--- |
| **랭크드** | 랭킹에 반영되는 경쟁 모드. 설정 변경 불가. |
| **캐주얼** | 랭킹에 반영되지 않는 일반 모드. 일부 설정 변경 가능. |
| **커스텀** | 모든 설정을 자유롭게 변경할 수 있는 모드. |

### 3.1.3. AI 난이도 상세

| 난이도 | AI 특성 |
| :--- | :--- |
| **쉬움** | 수동적 AI. 공격을 거의 하지 않음. 방어 위주. |
| **보통** | 균형잡힌 AI. 기회가 있을 때만 공격. |
| **어려움** | 공격적 AI. 적극적으로 확장하고 공격함. |

## 3.2. 국가 및 도시 선택

### 3.2.1. 할당 방식

| 방식 | 설명 |
| :--- | :--- |
| **수동 선택** | 플레이어가 원하는 국가와 도시를 직접 선택 |
| **랜덤 할당** | 시스템이 무작위로 국가와 도시를 배분 |
| **밸런스 할당** | 초기 자원과 위치를 고려하여 공평하게 배분 |

### 3.2.2. 국가 선택 화면

플레이어가 방에 입장하면 다음과 같은 국가 선택 화면이 표시됩니다:

```
선택 가능한 국가 목록
한국 (3/6 선택됨)
일본 (2/8 선택됨)
중국 (4/15 선택됨)
러시아 (0/8 선택됨)
기타... (나머지 국가)
```

### 3.2.3. 도시 선택 화면

국가를 선택한 후, 해당 국가 내에서 도시를 선택합니다:

```
한국의 도시 목록
서울 (수도, 선택됨)
부산 (주요도시, 선택가능)
대구 (일반도시, 선택가능)
대전 (일반도시, 선택가능)
광주 (일반도시, 선택가능)
인천 (주요도시, 선택가능)
```

### 3.2.4. AI 자동 할당

플레이어가 선택하지 않은 국가와 도시는 모두 AI가 할당받습니다. AI는 난이도별로 초기 전략이 설정됩니다.

## 3.3. 게임 중 이탈 및 재접속 처리

### 3.3.1. 플레이어 이탈 시

플레이어가 게임 중 이탈(네트워크 끊김 또는 의도적 종료)할 경우:

| 경과 시간 | 처리 |
| :--- | :--- |
| 즉시 | 해당 영주의 턴 액션이 자동 스킵됨 |
| 3턴 후 | AI가 해당 영주를 자동 조종 시작 |
| 재접속 시 | 기존 영주로 복구 가능 |

### 3.3.2. 모든 플레이어 이탈 시

방에 접속한 플레이어가 0명이 되면:

| 항목 | 상태 |
| :--- | :--- |
| 게임 진행 | 일시 정지 |
| 턴 타이머 | 멈춤 |
| 자원 생산 | 중단 |
| AI 활동 | 중단 |

새 플레이어가 입장하면 게임이 재개됩니다.

## 3.4. 게임 오버 및 관전/AI 조종 모드

### 3.4.1. 게임 오버 조건

플레이어의 마지막 도시가 함락되면 게임 오버가 됩니다. 해당 영주는 리더보드에서 제거됩니다.

### 3.4.2. 게임 오버 후 선택지

게임 오버가 된 플레이어는 다음 중 하나를 선택할 수 있습니다:

| 선택지 | 설명 |
| :--- | :--- |
| **관찰자 모드** | 게임을 계속 시청만 가능. 채팅 불가. 뉴스/지도 볼 수 있음. |
| **AI 조종 모드** | 아직 생존한 다른 AI 영주를 조종 가능. 게임 중간부터 플레이 가능. |
| **퇴장** | 방을 완전히 나감. |

## 3.5. 방 비활성화 및 자동 삭제 정책

### 3.5.1. 비활성 상태

방에 접속한 플레이어가 0명일 경우, 해당 방은 즉시 비활성 상태로 변경됩니다.

### 3.5.2. 자동 삭제

비활성 상태가 **7일(168시간)** 지속될 경우, 해당 방의 모든 데이터(DB)는 영구 삭제됩니다.

### 3.5.3. 활성 방 제한

한 플레이어가 동시에 생성할 수 있는 활성 방은 최대 3개까지입니다.

# 4장. 월드 맵 시스템

## 4.1. 맵 구조: 육각형 타일 그리드 (Hex Grid)

### 4.1.1. 기본 구조

본 게임의 월드 맵은 문명(Civilization) 시리즈와 유사한 **육각형 타일 그리드(Hex Grid)** 시스템을 채택합니다. 육각형 타일은 사각형 타일에 비해 더 자연스러운 이동 경로와 균등한 인접 타일 수(6개)를 제공하여 전략적 깊이를 높입니다.

### 4.1.2. 맵 규모

| 항목 | 수치 |
| :--- | :---: |
| 총 타일 수 | 약 1,000개 이상 |
| 도시 수 | 100개 |
| 국가 수 | 20개 |
| 도시당 타일 수 | 7개 (중앙 + 주변 6개) |
| 자연 지형 타일 | 약 300개 |

### 4.1.3. 줌 레벨

플레이어는 맵을 확대/축소하여 볼 수 있습니다:

| 줌 레벨 | 배율 | 용도 |
| :--- | :---: | :--- |
| **축소** | 75% | 전체 맵 조망, 전략적 상황 파악 |
| **기본** | 100% | 일반적인 플레이 |
| **확대** | 150% | 세부 정보 확인, 전투 상황 파악 |

### 4.1.4. 스크롤

맵은 무제한 스크롤이 가능하며, 가장자리 래핑(지구본처럼 연결)은 적용되지 않습니다.

# 5장. 타일 및 지형 시스템

## 5.1. 도시 클러스터: 중앙 타일 + 주변 6타일 구조

### 5.1.1. 도시 타일 구조

하나의 도시는 **7개의 타일 묶음**으로 구성됩니다:

```
        [N]
    [NW]   [NE]
       [C]
    [SW]   [SE]
        [S]

C: 중앙 타일 (도시 본체, 궁전/시청 위치)
N: 북쪽 타일
NE: 북동쪽 타일
SE: 남동쪽 타일
S: 남쪽 타일
SW: 남서쪽 타일
NW: 북서쪽 타일
```

### 5.1.2. 중앙 타일 (Center)

중앙 타일은 도시의 핵심입니다:

| 특성 | 설명 |
| :--- | :--- |
| **건물** | 궁전/시청이 위치 (파괴 불가) |
| **점령 기준** | 이 타일을 점령해야 도시 소유권 이전 |
| **병력 상주** | 기본 500명 상주 (궁전 방어) |
| **건설 불가** | 추가 건물 건설 불가 |

### 5.1.3. 주변 6타일

주변 6개 타일은 건물 건설 및 병력 배치가 가능합니다:

| 특성 | 설명 |
| :--- | :--- |
| **건물 건설** | 도시 등급에 따라 타일당 1~3개 건물 건설 가능 |
| **병력 배치** | 타일당 최대 500명 배치 가능 |
| **지형 효과** | 각 타일마다 다른 지형이 적용됨 |
| **방어선 구축** | 적의 침입을 막는 방어선 역할 |

## 5.2. 지형 종류 (8종)

### 5.2.1. 지형 목록

본 게임에는 8가지 지형이 존재합니다:

| 지형 | 이동 비용 | 방어 보너스 | 경제 효과 | 특수 효과 |
| :--- | :---: | :---: | :--- | :--- |
| **평야** | 1.0 | 0% | 식량 생산 +10% | 건설 및 이동 표준 |
| **초원** | 0.7 | 0% | - | 기병 이동력 보너스 |
| **산악** | 2.0 | +20% | 병력 생산 -10% | 공성병기 진입 불가 |
| **언덕** | 1.5 | +10% | - | 궁병 사거리/공격 보너스 |
| **숲** | 1.2 | +5% | 정찰 성공률 -20% | 매복 전략 유리 (기습) |
| **산림** | 2.0 | +10% | 정찰 불가 (숨김) | 깊은 숲, 시야 차단 |
| **사막** | 1.3 | +15% | 식량 생산 -15% | 소모전 유도 |
| **해안/바다** | 1.0 | 0% | 돈 생산 +20% (항구 시) | 해군/수송선만 이동 가능 |

### 5.2.2. 지형별 상세 설명

**평야 (Plains)**

평야는 가장 기본적인 지형입니다. 이동과 건설에 페널티가 없으며, 식량 생산에 약간의 보너스가 있습니다. 대부분의 수도와 주요 도시는 평야에 위치합니다.

**초원 (Grassland)**

초원은 이동이 빠른 지형입니다. 특히 기병의 이동력에 보너스가 적용되어, 기병 중심의 전략에 유리합니다. 방어 보너스는 없습니다.

**산악 (Mountain)**

산악은 방어에 매우 유리한 지형입니다. 방어력 +20% 보너스가 적용되며, 공성병기는 산악 지형에 진입할 수 없습니다. 단, 병력 생산에 -10% 페널티가 있습니다.

**언덕 (Hill)**

언덕은 중간 정도의 방어 보너스를 제공합니다. 특히 궁병이 언덕에 배치되면 사거리와 공격력에 보너스가 적용됩니다.

**숲 (Forest)**

숲은 은폐에 유리한 지형입니다. 정찰 성공률이 -20% 감소하며, 매복 전략을 사용할 때 기습 보너스가 적용됩니다.

**산림 (Deep Forest)**

산림은 숲보다 더 깊은 숲입니다. 정찰이 완전히 불가능하며, 시야가 차단됩니다. 방어 보너스도 숲보다 높습니다.

**사막 (Desert)**

사막은 식량 생산에 불리하지만, 방어에는 유리한 지형입니다. 소모전을 유도하는 데 적합합니다.

**해안/바다 (Coast/Sea)**

해안과 바다는 해군만 이동할 수 있는 지형입니다. 항구가 있는 도시에서만 바다 타일로 진입할 수 있습니다. 항구가 있으면 돈 생산에 +20% 보너스가 적용됩니다.

## 5.3. 지형별 전투 효과

### 5.3.1. 공격 시 지형 효과

| 지형 | 공격력 보정 | 비고 |
| :--- | :---: | :--- |
| 평야 | +20% | 기동 좋음 |
| 초원 | +25% (기병) | 기병 특화 |
| 산악 | -15% | 접근 어려움 |

### 6.2.1. 도시 등급 분류

도시는 4가지 등급으로 분류됩니다:

| 등급 | 도시 수 | 타일당 건물 한도 | 초기 병력 | 초기 돈 | 초기 식량 | 점수 |

| **작은 마을** | 8개 | 1개 | 600 | 1,500 | 1,000 | -10 |

### 6.2.2. 등급별 특징

**수도 (Capital)**

수도는 각 국가의 정치적, 경제적 중심지입니다. 가장 많은 초기 자원과 병력을 보유하며, 타일당 최대 3개의 건물을 건설할 수 있습니다. 수도에만 건설할 수 있는 특수 건물(전략 사령부, 첩보 본부)이 있습니다.

**주요 도시 (Major City)**

주요 도시는 경제적, 문화적으로 중요한 도시입니다. 수도 다음으로 많은 자원을 보유하며, 타일당 최대 2개의 건물을 건설할 수 있습니다. 주요 도시에만 건설할 수 있는 특수 건물(국제 무역소)이 있습니다.

**일반 도시 (Normal City)**

일반 도시는 표준적인 도시입니다. 적당한 자원을 보유하며, 타일당 1개의 건물만 건설할 수 있습니다.

**작은 마을 (Small Town)**

작은 마을은 소규모 정착지입니다. 자원이 적고, 점수에서 오히려 -10점의 페널티가 있습니다. 하지만 전략적 요충지에 위치한 경우가 많아, 확장의 발판이 될 수 있습니다.

## 6.3. 도시 동적 성장 시스템

### 6.3.1. City Score (도시 점수)

도시는 게임 진행 중 성장하여 등급이 상승할 수 있습니다. 도시 점수(City Score)는 다음과 같이 계산됩니다:

```
City Score = 인구 + (건물 수 × 10) + (턴당 총 생산량 / 10)
```

### 6.3.2. 등급 승격 조건

| 현재 등급 | 승격 조건 | 승격 후 등급 |
| :--- | :--- | :--- |
| 작은 마을 | City Score 500 이상 | 일반 도시 |
| 일반 도시 | City Score 1,000 이상 | 주요 도시 |
| 주요 도시 | City Score 2,000 이상 + 수도 함락 | 수도 |

### 6.3.3. 수도 이전

원래 수도가 함락되면, 해당 국가의 가장 높은 City Score를 가진 주요 도시가 새로운 수도가 됩니다. 수도가 이전되면 기존 수도 전용 건물은 파괴되고, 새 수도에 건설해야 합니다.

## 6.4. 내전 처리

### 6.4.1. 내전 정의

같은 국가 내의 플레이어들이 서로 공격하는 상황을 내전(Civil War)이라고 합니다.

### 6.4.2. 내전 처리 방식

내전이 발생하면 해당 플레이어는 **새로운 국가로 취급**됩니다:

| 항목 | 처리 방식 |
| :--- | :--- |
| **국가명** | 새로 설정 가능 (예: 남한, 북한) |
| **외교 관계** | 원래 국가와 별개로 처리 |
| **수도 버프** | 내전 선포 시 공유 중단 |
| **동맹** | 원래 국가의 동맹 관계 해제 |

### 6.4.3. 내전 선포

내전을 선포하려면 다음 조건을 충족해야 합니다:

| 조건 | 설명 |
| :--- | :--- |
| **도시 수** | 최소 2개 이상의 도시 보유 |
| **병력** | 최소 3,000명 이상의 병력 보유 |
| **비용** | 돈 5,000 소모 |

# 7장. 자원 시스템

## 7.1. 4대 핵심 자원

### 7.1.1. 자원 개요

본 게임에는 4가지 핵심 자원이 존재합니다:

| 자원 | 역할 | 획득 방법 | 소비 방법 |
| :--- | :--- | :--- | :--- |
| **병력 (Troops)** | 전투 및 점령의 기본 단위 | 도시 생산, 징집 | 전투 손실, 유지비 |
| **돈 (Gold)** | 모든 거래 및 건설의 화폐 | 도시 생산, 거래 | 건설, 거래, 정찰 |
| **식량 (Food)** | 병력 유지 및 인구 성장 | 도시 생산, 거래 | 병력 유지비, 인구 유지 |
| **특산물 (Specialty)** | 행복도 및 거래의 핵심 | 도시 생산, 거래 | 소비, 거래 |

### 7.1.2. 병력 (Troops)

병력은 전투와 점령의 기본 단위입니다.

| 항목 | 수치 |
| :--- | :--- |
| **초기값** | 600~2,000명 (도시 등급별) |
| **최대값** | 도시 인구의 100% |
| **안전 징집량** | 도시 인구의 10% |
| **생산량** | 도시 기본 5~20명/턴 |
| **유지비** | 100명당 식량 7/턴 |
| **타일당 최대 배치** | 500명 |

**과도한 징집의 페널티**

도시 인구의 10%를 초과하여 징집하면 행복도가 감소합니다:

| 징집 비율 | 행복도 감소 |
| :--- | :---: |
| 10~20% | -5% |
| 20~30% | -10% |
| 30~50% | -20% |
| 50% 이상 | -30% + 반란 위험 |

**반란 시 병력 처리**

도시가 반란하면 해당 도시의 모든 병력은 적이 됩니다. 병력이 0명인 상태에서 반란이 발생하면, 해당 도시 인구의 50%가 병력으로 전환되어 반란군이 됩니다. 반란군은 도시 타일의 지형에 따라 보병/기병/궁병/해군으로 분배됩니다.

### 7.1.3. 돈 (Gold)

돈은 모든 거래와 건설의 화폐입니다.

| 항목 | 수치 |
| :--- | :--- |
| **초기값** | 1,500~8,000 (도시 등급별) |
| **최대값** | 무제한 |

### 7.1.4. 식량 (Food)

식량은 병력 유지와 인구 성장에 필수적인 자원입니다.

| 항목 | 수치 |

| 상황 | 페널티 |

# 8장. 생산 시스템

## 8.1. 턴별 최종 생산량 공식

### 8.1.1. 기본 공식

모든 자원의 최종 생산량은 다음 공식으로 계산됩니다:

```
최종 생산량 = 기본값 × (1 + 건물 보너스) × (1 + 지형 보너스) × 행복도 배율
```

### 8.1.2. 건물 보너스 (Building Bonus)

해당 도시에 건설된 모든 생산 관련 건물의 효과를 합산합니다.

예시: 시장(+40%) + 조세청(+50%) = 돈 생산 +90%

### 8.1.3. 지형 보너스 (Terrain Bonus)

도시 중심 타일의 지형에 따라 기본 생산량에 가산됩니다.

| 지형 | 병력 | 돈 | 식량 |
| :--- | :---: | :---: | :---: |
| 평야 | +10% | +5% | +10% |
| 산악 | -10% | 0% | -5% |
| 해안 | 0% | +20% | 0% |
| 사막 | -15% | 0% | -15% |

### 8.1.4. 행복도 배율 (Happiness Multiplier)

국민 행복도(0~100%)는 모든 생산량에 직접적인 배율로 작용합니다.

| 행복도 (%) | 생산 배율 | 특이사항 |
| :--- | :---: | :--- |
| **90~100** | 1.15 | 생산량 +15% (황금기) |
| **70~89** | 1.00 | 표준 생산량 |
| **50~69** | 0.85 | 생산량 -15% |
| **30~49** | 0.60 | 생산량 -40% (파업) |
| **0~29** | 0.30 | 생산량 -70%, 반란 위험 |

## 8.2. 도시 등급별 기본 생산량 (밸런스 조정 완료)

### 8.2.1. 조정된 기본 생산량

| 도시 등급 | 병력 (명/턴) | 돈 (Gold/턴) | 식량 (Food/턴) | 특산물 (Unit/턴) |
| :--- | :---: | :---: | :---: | :---: |
| **수도** | 20 | 150 | **120** | 15 |
| **주요 도시** | 15 | 100 | **90** | 10 |
| **일반 도시** | 10 | 70 | **60** | 7 |
| **작은 마을** | 5 | 40 | **30** | 4 |

(식량 생산량은 기존 대비 50% 상향 조정됨)

### 8.2.2. 생산량 계산 예시

서울(수도)의 턴당 돈 생산량 계산:

```
기본값: 150
건물 보너스: 시장(+40%) + 조세청(+50%) = +90%
지형 보너스: 평야(+5%)
행복도: 80% → 배율 1.0

최종 생산량 = 150 × (1 + 0.9) × (1 + 0.05) × 1.0
            = 150 × 1.9 × 1.05 × 1.0
            = 299.25 ≈ 299 돈/턴
```

## 8.3. 건물 및 지형 보너스 상세

### 8.3.1. 돈 생산 관련 건물

| 건물 | 돈 생산 보너스 | 기타 효과 |
| :--- | :---: | :--- |
| 시장 | +40% | - |

# 10장. 시민 행복도 시스템

## 10.1. 행복도 결정 공식

### 10.1.1. 기본 공식

행복도는 다음 공식으로 계산됩니다:

```
행복도 = 기본값(50) + 증가 요인 - 감소 요인
```

### 10.1.2. 행복도 범위

행복도는 0~100% 범위로 제한됩니다.

## 10.2. 행복도 증가 요인

| 차 | +6% |
| 과일 | +6% |
| 술 | +5% |
| 향신료 | +8% |

### 10.2.2. 문화 건물

| :--- | :---: |
| 공원/극장 | +10% |
| 병원 | +5% |

### 10.2.3. 기타 증가 요인

| 요인 | 행복도 증가 |
| :--- | :---: |
| 전투 승리 | +3% (일시적) |
| 거래 성공 | +2% (일시적) |
| 평화 상태 (5턴 이상) | +5% |
| 세금 감면 | +5~10% |

## 10.3. 행복도 감소 요인

### 10.3.1. 세율

| 세율 | 행복도 감소 |
| :--- | :---: |
| 저세율 (50%) | 0% |
| 표준 (100%) | -5% |
| 고세율 (150%) | -15% |

### 10.3.2. 전쟁 피로

| 상황 | 행복도 감소 |
| :--- | :---: |

# 11장. 병과 시스템

## 11.1. 병과 개요

### 11.1.1. 병과 시스템 설계 철학

본 게임의 병과 시스템은 복잡한 유닛 생산이 아닌, 징집된 병력에 장비/훈련을 입히는 추상화 방식을 사용합니다. 플레이어는 병력을 징집한 후, 해당 병력의 병과 비율을 설정하여 전투에 투입합니다.

### 11.1.2. 병과 목록

본 게임에는 6가지 병과가 존재합니다:

| 병과 | 역할 | 필요 조건 |
| :--- | :--- | :--- |
| **보병** | 범용 전투, 점령 | 없음 |
| **기병** | 기동전, 측면 공격 | 말 특산물 |
| **궁병** | 원거리 공격, 방어 | 없음 |
| **공성병기** | 건물 파괴, 공성전 | 무기/화약 특산물 |
| **해군** | 해상 이동, 해전 | 항구 건물 |
| **첩보** | 정보 수집, 공작 | 외교관저 건물 |

| 공격 병과 | 유리한 상대 | 불리한 상대 |

### 11.3.2. 상성 적용 예시

기병 500명이 보병 500명을 공격하는 경우:

```
기본 공격력: 500 × 1.5 = 750
상성 보너스: 750 × 1.2 = 900 (보병에게 +20%)

```

## 12.1. 전투 개요

### 12.1.1. 전투 시스템 설계 철학

본 게임의 전투 시스템은 **순수 LLM 텍스트 판정 방식**을 채택합니다. 플레이어는 정해진 프리셋이나 태세 선택 없이, **자유로운 텍스트로 전략을 직접 작성**합니다. LLM은 이 전략을 분석하여 전략 점수를 부여하고, 이 점수가 전투 결과에 반영됩니다. 전략을 입력하지 않은 경우 **전략 점수 0점**으로 처리되어 능력치만으로 전투가 진행됩니다.

### 12.1.2. 전투력 구성 비율

전투력은 다음 비율로 구성됩니다:

| 요소 | 비중 |
| :--- | :---: |
| **능력치 점수** | 70% |
| **전략 점수** | 30% |

## 12.2. 전투 프로세스

### 12.2.1. 전투 4단계 프로세스

**[1단계] 공격 선언**

공격자가 대상 타일을 지정하고 병력을 투입합니다.

```
공격 선언 예시:
- 공격 도시: 서울
- 목표 타일: 도쿄 북쪽 타일
- 투입 병력: 1,000명 (보병 500, 기병 300, 궁병 200)
```

**[2단계] 방어 준비**

방어자에게 공격 알림이 전송됩니다. 방어자는 해당 턴 내에 방어 전략을 수립해야 합니다.

```
방어 알림 예시:
"서울이 당신의 도쿄 북쪽 타일을 공격하려고 합니다!"
- 예상 공격 병력: 약 1,000명 (첩보력에 따라 정확도 다름)
- 방어 준비 시간: 현재 턴 종료까지
```

**[3단계] 전략 입력**

공격자와 방어자가 동시에 전략을 입력합니다.

**[4단계] 판정 및 결과**

LLM이 양측의 전략을 분석하고 전투 결과를 판정합니다.

### 12.2.2. 전략 입력 시스템

전략 입력은 **자유 텍스트 방식**으로 진행됩니다. 태세 선택이나 프리셋 없이, 플레이어가 직접 전략을 작성합니다.

**전략 텍스트 입력 규칙**

| 항목 | 규칙 |
| :--- | :--- |
| 글자 수 | 200자 이내 |
| 입력 시간 | 턴 종료 전까지 |
| 미입력 시 | **전략 점수 0점** 처리 |
| 언어 | 한국어/영어 모두 가능 |

**전략 텍스트 작성 가이드**

좋은 전략 텍스트는 다음 요소를 포함해야 합니다:

| 요소 | 설명 | 예시 |
| :--- | :--- | :--- |
| **지형 활용** | 전투 타일의 지형을 어떻게 활용할 것인가 | "숲의 엄폐물을 활용하여..." |
| **병과 배치** | 각 병과를 어디에 어떻게 배치할 것인가 | "기병은 우측 측면에, 궁병은 후방 언덕에..." |
| **전술 목표** | 무엇을 달성하려고 하는가 | "적의 보급로를 차단하여..." |
| **상황 대응** | 예상되는 적의 행동에 어떻게 대응할 것인가 | "적이 측면 공격 시 예비대로 대응..." |

**공격자 전략 예시**

```
"기병 300을 우측 숲에 매복시키고, 보병 500으로 정면에서 적의 주의를 끈다.
적이 정면에 집중하면 기병이 측면을 타격하고, 궁병 200은 후방 언덕에서 지원 사격한다.
목표는 적의 지휘부를 혼란에 빠뜨려 조직적 방어를 무력화하는 것이다."
```

**방어자 전략 예시**

```
"성벽 위에 궁병 400을 배치하여 접근하는 적에게 원거리 공격을 집중한다.
보병 300은 성문 앞에서 적의 돌파를 저지하고, 기병 200은 예비대로 측면 기습에 대비한다.
첩보에 따르면 적이 기습을 준비 중이므로, 숲 방향에 정찰병을 배치한다."
```

**미입력 시 처리**

전략 텍스트를 입력하지 않으면 **전략 점수 0점**으로 처리됩니다. 이 경우 능력치 점수만으로 전투가 진행되며, 상대가 전략을 입력했다면 최대 30%의 불이익을 받게 됩니다.

## 12.3. 전투력 계산

### 12.3.1. 능력치 점수 (70%)

능력치 점수는 다음 공식으로 계산됩니다:

```
능력치 점수 = 병력수 × 병과계수 × (1 + 건물보너스) × (1 + 지형보너스) × (1 + 상성보너스) × 행복도배율
```

**공격력 계산 예시**

서울(공격자)이 도쿄(방어자)를 공격하는 경우:

```
병력: 1,000명 (보병)
병과 계수: 1.0
건물 보너스: 무기고 +20% = 0.2
지형 보너스: 평야 +20% = 0.2
상성 보너스: 없음 = 0
행복도: 70% → 배율 1.0

능력치 점수 = 1,000 × 1.0 × (1 + 0.2) × (1 + 0.2) × (1 + 0) × 1.0
            = 1,000 × 1.0 × 1.2 × 1.2 × 1.0 × 1.0
            = 1,440
```

### 12.3.2. 전략 점수 (30%)

전략 점수는 LLM이 부여하는 0~30점 사이의 점수입니다.

**전략 점수 평가 기준**

| 평가 항목 | 배점 | 상세 판정 기준 |
| :--- | :---: | :--- |
| **지형 일치성** | 10점 | 전략이 현재 전투 타일의 지형을 효과적으로 활용하는가? |
| **병과 연계성** | 5점 | 투입된 병력의 병과와 전략이 논리적으로 연결되는가? |
| **병법 논리성** | 10점 | 포위, 기습, 보급로 차단 등 실제 병법의 원리가 적용되었는가? |
| **첩보 카운터** | 5점 | (방어자 한정) 첩보를 통해 얻은 상대 전략에 대한 직접적인 대응을 제시했는가? |

**전략 점수 적용**

전략 점수는 다음과 같이 전투력에 반영됩니다:

```
전투력 보정 계수 = 1 + (전략 점수 / 100)
```

전략 점수 30점 만점 시, 전투력에 1.3배(30%) 가중치가 부여됩니다.

### 12.3.3. 최종 전투력 계산

```
최종 전투력 = 능력치 점수 × 전투력 보정 계수
```

**계산 예시**

```
능력치 점수: 1,440
전략 점수: 20점 → 보정 계수 1.2

최종 전투력 = 1,440 × 1.2 = 1,728
```

## 12.4. 전투 결과 판정

### 12.4.1. 승패 판정

양측의 최종 전투력을 비교하여 승패를 판정합니다.

```
승률 = 공격자 전투력 / (공격자 전투력 + 방어자 전투력) × 100
최종 승률 = 기본 승률 ± 무작위 요소(±10%)
```

### 12.4.2. 병력 손실 계산

전투력 차이에 따라 병력 손실이 결정됩니다.

| 전투력 차이 | 승자 손실 | 패자 손실 |
| :--- | :---: | :---: |
| 압도적 승리 (2배 이상) | 10~20% | 80~100% |
| 대승 (1.5~2배) | 20~30% | 60~80% |
| 신승 (1.2~1.5배) | 30~40% | 50~60% |
| 박빙 (1~1.2배) | 40~50% | 50~60% |

### 12.4.3. 타일 점령 조건

타일을 점령하려면 다음 조건을 충족해야 합니다:

| 조건 | 설명 |
| :--- | :--- |
| **적 병력 0명** | 해당 타일의 적 병력이 모두 제거됨 |
| **아군 병력 100명 이상** | 해당 타일에 아군 병력 100명 이상 진입 |

### 12.4.4. 도시 점령

도시 중앙 타일(궁전/시청)을 점령하면 해당 도시의 소유권이 이전됩니다. 도시를 점령하면 해당 도시가 소유한 모든 타일도 함께 점령됩니다.

### 12.4.5. 약탈

타일 점령 시 다음과 같은 약탈이 발생합니다:

| 항목 | 약탈량 |
| :--- | :--- |
| 돈 | 20~30% |
| 특산물 | 10~20% |
| 건물 파괴 | 20% 확률 |

# 13장. LLM 전투 판정 상세

## 13.1. LLM 판정 프로세스

### 13.1.1. 판정 흐름

```
1. 양측 전략 텍스트 수집
2. 전투 컨텍스트 구성 (지형, 병력, 건물 등)

5. 전투 결과 계산

```

### 13.1.2. LLM API 호출 데이터 구조

LLM API 호출 시 다음과 같은 JSON 형식의 데이터를 전송합니다:

```
{
  "battle_id": "unique_battle_id",
  "terrain": "Forest",
  "weather": "Clear",
  "attacker": {
    "player_id": "player1",

    "units": [
      {"type": "Cavalry", "count": 300},
      {"type": "Infantry", "count": 500},
      {"type": "Archer", "count": 200}
    ],
    "buildings": ["Armory", "Training_Ground"],
    "intelligence_level": 3,
    "happiness": 75
  },
  "defender": {
    "player_id": "player2",

    "units": [
      {"type": "Archer", "count": 400},
      {"type": "Infantry", "count": 300},
      {"type": "Cavalry", "count": 200}
    ],
    "buildings": ["Wall", "Fortress"],
    "intelligence_level": 2,
    "happiness": 80
  }
}
```

**전략 미입력 시 데이터 구조**

전략을 입력하지 않은 경우, `strategy_text` 필드가 빈 문자열로 전송되고 해당 플레이어의 전략 점수는 **0점**으로 처리됩니다.

```
{
  "attacker": {
    "strategy_text": "",  // 미입력 시 빈 문자열
    ...
  }
}
```

## 13.2. 전략 점수 산출 상세

### 13.2.1. 지형 일치성 (10점)

| 점수 | 기준 |
| :---: | :--- |
| 10점 | 지형을 완벽하게 활용한 전략 (숲에서 매복, 산악에서 고지 점령 등) |
| 7점 | 지형을 적절히 활용한 전략 |
| 4점 | 지형을 고려했으나 활용이 미흡한 전략 |
| 1점 | 지형을 무시한 전략 |
| 0점 | 지형에 반하는 전략 (사막에서 기병 돌격 등) |

### 13.2.2. 병과 연계성 (5점)

| 점수 | 기준 |
| :---: | :--- |
| 5점 | 병과 특성을 완벽하게 활용한 전략 |
| 3점 | 병과 특성을 적절히 활용한 전략 |
| 1점 | 병과 특성을 고려하지 않은 전략 |
| 0점 | 병과 특성에 반하는 전략 (궁병으로 돌격 등) |

### 13.2.3. 병법 논리성 (10점)

| 점수 | 기준 |
| :---: | :--- |
| 10점 | 고급 병법 원리 적용 (포위, 분산, 집중, 기만 등) |
| 7점 | 기본 병법 원리 적용 |
| 4점 | 논리적이나 평범한 전략 |
| 1점 | 비논리적인 전략 |
| 0점 | 실행 불가능한 전략 |

### 13.2.4. 첩보 카운터 (5점, 방어자 한정)

| 점수 | 기준 |
| :---: | :--- |
| 5점 | 첩보로 얻은 정보를 완벽하게 활용한 대응 |
| 3점 | 첩보 정보를 적절히 활용한 대응 |
| 1점 | 첩보 정보를 언급했으나 활용이 미흡 |
| 0점 | 첩보 정보를 활용하지 않음 |

# 14장. 점령 및 약탈 시스템

## 14.1. 타일 점령

타일을 점령하려면 다음 조건을 모두 충족해야 합니다:

| 조건 | 설명 |

### 14.2.2. 도시 점령 시 처리

도시 점령 시 다음과 같이 처리됩니다:

| 항목 | 처리 |
| :--- | :--- |

### 14.3.3. 약탈 방지

다음 조건에서는 약탈이 감소합니다:

| 조건 | 약탈 감소 |

# 15장. 건물 시스템

## 15.1. 건물 시스템 개요

### 15.1.1. 설계 철학

본 게임의 건물 시스템은 v8.0의 20종 건물을 채택하되, v4.0의 수리/파괴 메커니즘을 적용합니다. 건물은 도시의 경제, 군사, 정보 능력을 강화하는 핵심 요소입니다.

### 15.1.2. 건물 분류

건물은 4가지 카테고리로 분류됩니다:

| 카테고리 | 건물 수 | 주요 역할 |
| :--- | :---: | :--- |
| **군사 시설** | 5종 | 병력 생산, 공격력/방어력 증가 |
| **경제 시설** | 5종 | 자원 생산 증가 |
| **정보/외교 시설** | 5종 | 첩보력 증가, 첩보 활동 지원 |
| **국가/행복도 시설** | 5종 | 행복도 증가, 특수 효과 |

## 15.2. 건설 규칙

### 15.2.1. 건설 위치

건물은 도시 중앙 타일을 제외한 주변 6개 타일에 건설할 수 있습니다.

### 15.2.2. 타일당 건설 한도

도시 등급에 따라 타일당 건설 가능한 건물 수가 다릅니다:

| 도시 등급 | 타일당 최대 건물 수 |
| :--- | :---: |
| 수도 | 3개 |
| 주요 도시 | 2개 |
| 일반 도시 | 1개 |
| 작은 마을 | 1개 |

### 15.2.3. 건설 제한

일부 건물은 특정 조건에서만 건설할 수 있습니다:

| 건물 | 건설 제한 |
| :--- | :--- |
| 전략 사령부 | 수도에만 건설 가능, 국가당 1개 |
| 첩보 본부 | 수도에만 건설 가능 |
| 국제 무역소 | 주요 도시 이상에만 건설 가능 |
| 항구 | 해안 타일에만 건설 가능 |

### 15.2.4. 건설 프로세스

1. 건설할 건물과 타일 선택
2. 건설 비용(돈) 지불
3. 건설 기간(턴) 동안 대기
4. 건설 완료 후 효과 적용

전쟁 중인 상대에게 휴전을 제안할 수 있습니다:

| 항목 | 설명 |

| 비용 | 협상에 따라 다름 (돈, 자원, 영토 등) |
| 수락 시 | 양측 모두 중립 상태로 변경 |
| 휴전 기간 | 최소 10턴 동안 공격 불가 |

### 17.2.4. 항복

패배가 확실한 경우 항복을 선언할 수 있습니다:

| 항목 | 설명 |
| :--- | :--- |
| 효과 | 모든 도시와 자원을 상대에게 양도 |
| 약탈 감소 | 약탈량 -80% |
| 게임 오버 | 항복한 플레이어는 게임 오버 |

# 18장. 거래 시스템

## 18.1. 거래 개요

### 18.1.1. 거래 시스템 설계 철학

본 게임의 거래 시스템은 **PVP 직거래 방식**을 채택합니다. 플레이어 간 직접 협상을 통해 거래가 이루어지며, 시장 가격이나 수수료 개념은 적용되지 않습니다.

### 18.1.2. 거래 가능 품목

| 품목 | 거래 가능 여부 | 조건 |
| :--- | :---: | :--- |
| 돈 | 가능 | 제한 없음 |
| 식량 | 가능 | 제한 없음 |
| 특산물 | 가능 | 제한 없음 |
| **병력** | **가능** | 제한 없음 |
| **도시** | **가능** | 2개 이상 보유 시 |
| 건물 | 불가 | - |
| **평화협정** | **가능** | 전쟁 중일 경우 |
| **시야 공유** | **가능** | 제한 없음 |
| **첩보 포로** | **가능** | 포로 보유 시 |

## 18.2. 확장된 거래 품목 상세

### 18.2.1. 병력 거래

병력을 다른 플레이어에게 양도할 수 있습니다.

| 항목 | 설명 |
| :--- | :--- |
| **거래 단위** | 100명 단위 |
| **병과 지정** | 보병/기병/궁병/공성병기/해군 선택 가능 |
| **이동 시간** | 거래 체결 후 즉시 이전 |
| **제한** | 도시에 최소 100명 잔류 필요 |

**병력 거래 예시**

```
거래 제안:
- 제공: 보병 500명
- 요청: 돈 5,000 + 특산물(말) 50단위
```

### 18.2.2. 도시 거래

도시를 다른 플레이어에게 양도할 수 있습니다. 단, **2개 이상의 도시를 보유한 경우**에만 가능합니다.

| 항목 | 설명 |
| :--- | :--- |
| **거래 단위** | 도시 1개 |
| **제한** | 보유 도시 2개 이상 시만 가능 |
| **수도 거래** | 불가 (수도는 거래 불가) |
| **이전 시기** | 거래 체결 후 다음 턴부터 적용 |
| **포함 요소** | 도시 소유권, 소속 타일, 건물, 자원 모두 포함 |

**도시 거래 예시**

```
거래 제안:
- 제공: 부산 도시 (병력 1,200명, 건물 3개 포함)
- 요청: 돈 50,000 + 동맹 체결
```

### 18.2.3. 평화협정 거래

전쟁 중인 상대와 평화협정을 체결할 수 있습니다.

| 항목 | 설명 |
| :--- | :--- |
| **조건** | 전쟁 상태일 경우에만 가능 |
| **효과** | 전쟁 상태 해제, 외교 관계 "중립"으로 변경 |
| **휴전 기간** | 협정 후 10턴간 재선전포고 불가 |
| **위반 시** | 위반 국가 우호도 -50, 전체 국가 신뢰도 하락 |

**평화협정 예시**

```
평화협정 제안:
- 조건: 돈 10,000 지불 + 특산물(향신료) 100단위
- 추가 조건: 도시 1개 양도 (대구)
- 휴전 기간: 10턴
```

### 18.2.4. 시야 공유 협정

다른 플레이어와 시야를 공유하는 협정을 체결할 수 있습니다.

| 항목 | 설명 |
| :--- | :--- |
| **효과** | 상대방의 시야 범위 내 타일 정보 공유 |
| **기간** | 협정 시 지정 (10턴/30턴/무기한) |
| **해제** | 양측 합의 또는 일방적 해제 가능 |
| **일방 해제 시** | 우호도 -10 |

**시야 공유 예시**

```
시야 공유 협정 제안:
- 기간: 30턴
- 대가: 돈 3,000/턴 지불
```

### 18.2.5. 첩보 포로 교환

적국의 스파이를 생포한 경우, 해당 포로를 거래할 수 있습니다.

| 항목 | 설명 |
| :--- | :--- |
| **포로 획득** | 적 스파이 발각 시 사망 대신 생포 선택 가능 (50% 확률) |
| **포로 유지비** | 30 돈/턴 |
| **교환 대상** | 포로의 원래 소속 국가 |
| **교환 효과** | 포로 반환, 대가 획득 |

**첩보 포로 교환 예시**

```
포로 교환 제안:
- 제공: 중국 스파이 포로 1명
- 요청: 돈 5,000 + 우호도 +20
```

**포로 처형**

포로를 교환하지 않고 처형할 수도 있습니다.

| 항목 | 설명 |
| :--- | :--- |
| **처형 효과** | 포로 사망, 원래 국가와 우호도 -30 |
| **정보 획득** | 처형 전 심문으로 적 정보 일부 획득 가능 |

## 18.3. 거래 프로세스

### 18.3.1. 거래 제안

1. 거래 상대 선택
2. 제공할 품목 및 수량 입력 (다중 선택 가능)
3. 요청할 품목 및 수량 입력 (다중 선택 가능)
4. 추가 조건 입력 (평화협정, 시야공유 등)
5. 거래 제안 전송

### 18.2.2. 거래 수락/거절

거래 제안을 받은 플레이어는 다음 중 하나를 선택합니다:

| 선택 | 효과 |
| :--- | :--- |
| **수락** | 거래 즉시 체결, 품목 교환 |
| **거절** | 거래 취소 |
| **역제안** | 수정된 조건으로 재제안 |

### 18.2.3. 거래 체결

거래가 체결되면 다음과 같이 처리됩니다:

| 항목 | 처리 |
| :--- | :--- |
| 품목 교환 | 즉시 양측 자원에 반영 |
| 거래 기록 | 양측 거래 내역에 기록 |
| 우호도 | 양측 우호도 +2 |
| 행복도 | 양측 행복도 +2% (일시적) |

## 18.3. 특산물 가치 시스템

특산물은 종류에 따라 기본 가치가 다릅니다. 이 가치는 거래 시 참고용으로 제공됩니다.

특정 특산물을 독점하면 거래에서 유리한 위치를 점할 수 있습니다. 예를 들어, 말(군마)을 독점하면 기병을 원하는 국가에게 높은 가격을 요구할 수 있습니다.

### 18.4.2. 자원 봉쇄

적국의 자원 공급을 차단하여 경제적 압박을 가할 수 있습니다. 동맹국과 협력하여 특정 국가와의 거래를 금지하는 방식입니다.

# 19장. 첩보 시스템

## 19.1. 첩보 시스템 개요

### 19.1.1. 설계 철학

본 게임의 첩보 시스템은 v8.0의 스파이 시스템을 기반으로 하며, 정보 수집과 공작 활동을 통해 전략적 우위를 점하는 것을 목표로 합니다.

### 19.1.2. 첩보력 (Intelligence)

첩보력은 첩보 활동의 성공률에 영향을 미치는 수치입니다.

| 항목 | 수치 |
| :--- | :--- |
| 초기값 | 0 |
| 최대값 | 100 |
| 성장 | 턴당 +1 (대학 건물 시 +10%) |
| 효과 | 첩보 성공률 증가, 적 첩보 발각률 증가 |

## 19.2. 스파이 시스템

### 19.2.1. 스파이 생성

스파이는 외교관저 건물이 있는 도시에서 생성할 수 있습니다.

| 항목 | 수치 |
| :--- | :--- |
| 생성 비용 | 500 돈 |
| 생성 시간 | 2턴 |
| 유지비 | 50 돈/턴 |
| 최대 수 | 외교관저당 +1 |

### 19.2.2. 스파이 파견

스파이를 적 도시에 파견하여 첩보 활동을 수행합니다.

| 항목 | 설명 |
| :--- | :--- |
| 파견 비용 | 100 돈 |
| 이동 시간 | 거리에 따라 1~3턴 |
| 잠복 | 도착 후 자동으로 잠복 시작 |

### 19.2.3. 스파이 발각

스파이는 적에게 발각될 수 있습니다.

| 항목 | 확률 |
| :--- | :--- |
| 기본 발각률 | 3%/턴 |
| 첩보 본부 효과 | +3%/턴 (적 스파이 발각) |
| 발각 시 | 스파이 즉시 사망 |

## 19.3. 첩보 활동 종류

### 19.3.1. 정찰 (Reconnaissance)

적 도시의 정보를 수집합니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 50 돈 |
| 소요 시간 | 1턴 |
| 성공률 | 70% + (첩보력 × 0.3%) |
| 정보 | 병력 수, 건물 목록, 자원량 |

### 19.3.2. 전략 정찰 (Strategic Reconnaissance)

적의 전투 전략을 미리 파악합니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 200 돈 |
| 소요 시간 | 2턴 |
| 성공률 | 50% + (첩보력 × 0.3%) |
| 정보 | 적의 태세(Stance) 및 전략 텍스트 일부 |

### 19.3.3. 사보타주 (Sabotage)

적 도시의 건물을 파괴하거나 자원을 탈취합니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 300 돈 |
| 소요 시간 | 3턴 |
| 성공률 | 40% + (첩보력 × 0.3%) |
| 효과 | 건물 1개 파괴 또는 자원 20% 탈취 |

### 19.3.4. 선동 (Incitement)

적 도시의 행복도를 감소시킵니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 150 돈 |
| 소요 시간 | 2턴 |
| 성공률 | 60% + (첩보력 × 0.3%) |
| 효과 | 행복도 -10% (3턴 지속) |

### 19.3.5. 암살 (Assassination)

적 스파이를 제거합니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 400 돈 |
| 소요 시간 | 2턴 |
| 성공률 | 30% + (첩보력 × 0.3%) |
| 효과 | 적 스파이 1명 제거 |

### 19.3.6. 금 횡령 (Embezzlement)

적 도시의 돈을 탈취합니다.

| 항목 | 수치 |
| :--- | :--- |
| 비용 | 100 돈 |
| 소요 시간 | 3턴 |
| 성공률 | 50% + (첩보력 × 0.3%) |
| 효과 | 적 돈 10~20% 탈취 |
| 조건 | 국제 무역소 건물 필요 |

## 19.4. 첩보 방어

### 19.4.1. 방첩 활동

첩보 본부 건물이 있으면 자동으로 방첩 활동이 수행됩니다.

| 항목 | 효과 |
| :--- | :--- |
| 적 스파이 발각률 | +3%/턴 |
| 첩보 활동 방해 | 적 첩보 성공률 -10% |

### 19.4.2. 암호 해독국 효과

암호 해독국 건물이 있으면 추가 방어 효과가 적용됩니다.

| 항목 | 효과 |
| :--- | :--- |
| 전 도시 첩보 방어율 | +6% |
| 첩보 활동 가능 수 | +2 |

## 19.5. 첩보력 성장 시스템

### 19.5.1. 첩보력 성장 공식

```
턴당 첩보력 성장 = 1 + (대학 보너스) + (종이/문서 보너스)
```

### 19.5.2. 첩보력 성장 보너스

| 요인 | 보너스 |
| :--- | :--- |
| 대학 건물 | +10% |
| 종이/문서 특산물 | +20% |
| 첩보 활동 성공 | +2 (일회성) |

### 19.5.3. 첩보력 레벨별 효과

| 첩보력 | 효과 |
| :---: | :--- |
| 0~19 | 기본 첩보 활동만 가능 |
| 20~39 | 전략 정찰 가능 |
| 40~59 | 사보타주 가능 |
| 60~79 | 선동, 암살 가능 |
| 80~100 | 모든 첩보 활동 가능, 성공률 최대 |

# 21장. 뉴스 시스템

## 21.1. 뉴스 시스템 개요

### 21.1.1. 설계 철학

뉴스 시스템은 게임 내 주요 이벤트를 플레이어에게 알리고, 몰입감을 높이는 역할을 합니다. LLM을 활용하여 자연스러운 뉴스 문체로 이벤트를 전달합니다.

### 21.1.2. 뉴스 유형

| 유형 | 설명 |
| :--- | :--- |
| **전투 뉴스** | 전투 발생 및 결과 |
| **외교 뉴스** | 동맹, 선전포고, 휴전 등 |
| **경제 뉴스** | 대규모 거래, 자원 변동 |
| **첩보 뉴스** | 스파이 발각, 첩보 활동 결과 |
| **도시 뉴스** | 도시 점령, 반란, 건물 완성 |

## 21.2. 뉴스 생성 규칙

### 21.2.1. 전투 뉴스 생성

전투가 발생하면 다음 정보를 기반으로 뉴스가 생성됩니다:

| 정보 | 예시 |
| :--- | :--- |
| 공격자 | 서울 (한국) |
| 방어자 | 도쿄 (일본) |
| 결과 | 공격자 승리 |
| 병력 손실 | 공격자 300, 방어자 500 |

**뉴스 예시**:
> **[속보] 서울군, 도쿄 북쪽 타일 점령!**
> 한국의 서울군이 일본의 도쿄 북쪽 타일을 점령했습니다. 이번 전투에서 서울군은 300명의 병력을 잃었으나, 도쿄군은 500명의 병력을 잃었습니다. 전문가들은 이번 승리가 한국의 동아시아 패권 확립에 중요한 전환점이 될 것으로 분석하고 있습니다.

### 21.2.2. 외교 뉴스 생성

외교 이벤트가 발생하면 다음 정보를 기반으로 뉴스가 생성됩니다:

| 정보 | 예시 |
| :--- | :--- |
| 이벤트 | 동맹 체결 |
| 당사자 | 한국, 일본 |

**뉴스 예시**:
> **[외교] 한국-일본 동맹 체결!**
> 한국과 일본이 공식적으로 동맹을 체결했습니다. 양국은 공동 방어와 경제 협력을 약속했으며, 이번 동맹이 동아시아 지역의 세력 균형에 큰 영향을 미칠 것으로 예상됩니다.

### 21.2.3. 첩보 뉴스 생성

첩보 활동 결과가 발생하면 다음 정보를 기반으로 뉴스가 생성됩니다:

| 정보 | 예시 |
| :--- | :--- |
| 이벤트 | 스파이 발각 |
| 대상 | 중국 스파이 |
| 장소 | 서울 |

**뉴스 예시**:
> **[첩보] 서울에서 중국 스파이 발각!**
> 서울 당국이 중국 스파이를 발각하여 체포했습니다. 해당 스파이는 서울의 군사 정보를 수집하던 중 발각된 것으로 알려졌습니다. 이번 사건으로 한중 관계에 긴장이 고조되고 있습니다.

## 21.3. 뉴스 공개 범위

### 21.3.1. 공개 범위 분류

| 뉴스 유형 | 공개 범위 |
| :--- | :--- |
| 전투 뉴스 | 전체 공개 |
| 외교 뉴스 | 전체 공개 |
| 경제 뉴스 (대규모) | 전체 공개 |
| 경제 뉴스 (소규모) | 당사자만 |
| 첩보 뉴스 | 당사자만 |
| 도시 뉴스 | 전체 공개 |

### 21.3.2. 첩보력에 따른 추가 정보

첩보력이 높은 플레이어는 뉴스에서 추가 정보를 얻을 수 있습니다:

| 첩보력 | 추가 정보 |
| :---: | :--- |
| 0~29 | 기본 정보만 |
| 30~59 | 병력 규모 대략적 파악 |
| 60~89 | 병력 구성 파악 |
| 90~100 | 전략 일부 파악 |

# 22장. AI 영주 시스템

## 22.1. AI 영주 개요

### 22.1.1. AI 영주 역할

AI 영주는 플레이어가 선택하지 않은 도시를 관리합니다. AI 영주는 난이도에 따라 다른 전략을 사용합니다.

### 22.1.2. AI 난이도별 특성

| 난이도 | 공격성 | 외교 | 경제 |
| :--- | :--- | :--- | :--- |
| **쉬움** | 수동적 | 우호적 | 비효율적 |
| **보통** | 균형 | 중립적 | 표준 |
| **어려움** | 공격적 | 기회주의적 | 효율적 |

## 22.2. AI 의사결정 시스템

### 22.2.1. 의사결정 우선순위

AI 영주는 매 턴 다음 순서로 의사결정을 수행합니다:

1. 방어 필요 여부 확인
2. 자원 상태 확인
3. 건설 필요 여부 확인
4. 공격 기회 확인
5. 외교 기회 확인

### 22.2.2. 방어 의사결정

| 상황 | AI 행동 |
| :--- | :--- |
| 적 병력 인접 | 방어 태세 강화, 병력 집결 |
| 행복도 낮음 | 세율 감소, 문화 건물 건설 |
| 식량 부족 | 농장 건설, 거래 시도 |

### 22.2.3. 공격 의사결정

| 상황 | AI 행동 |
| :--- | :--- |
| 적 병력 열세 | 공격 시도 |
| 중립 타일 인접 | 확장 시도 |
| 동맹 요청 | 상황에 따라 수락/거절 |

## 22.3. AI 전략 텍스트 생성

### 22.3.1. AI 전략 템플릿

AI 영주도 LLM 전투 판정을 위해 전략 텍스트를 생성합니다. 난이도에 따라 전략의 품질이 달라집니다.

**쉬움 난이도 예시**:
> "병력을 모아 정면으로 공격한다."

**보통 난이도 예시**:
> "보병을 전면에 배치하고, 궁병으로 후방 지원한다."

**어려움 난이도 예시**:
> "기병을 우측 숲에 매복시켜 적의 측면을 타격한다. 보병은 정면에서 적의 주의를 끌고, 궁병은 언덕에서 원거리 공격으로 지원한다."

## C.1. 경제 밸런스 시뮬레이션

### C.1.1. 수도 경제 시뮬레이션 (100턴)

**초기 조건**:
- 도시 등급: 수도
- 초기 자원: 병력 2,000 / 돈 8,000 / 식량 5,000 / 특산물 500
- 건물: 없음 (궁전/시청 제외)
- 행복도: 70%

**턴별 생산량 (건물 없음)**:
- 병력: 20명/턴
- 돈: 150돈/턴
- 식량: 120식량/턴 (조정됨)
- 특산물: 15단위/턴

**턴별 소비량**:
- 병력 유지비: 2,000명 × 7/100 = 140식량/턴

**순 식량 생산**: 120 - 140 = **-20식량/턴** (적자)

**문제점**: 초기 상태에서 식량 적자 발생

**해결책**: 농장 건설 필수

### C.1.2. 농장 건설 후 시뮬레이션

**농장 건설 후 생산량**:
- 식량: 120 × 1.5 = 180식량/턴

**순 식량 생산**: 180 - 140 = **+40식량/턴** (흑자)

**결론**: 농장 1개 건설로 식량 안정화 가능

## C.2. 전투 밸런스 시뮬레이션

### C.2.1. 동등 병력 전투 시뮬레이션

**조건**:
- 공격자: 보병 1,000명, 전략 점수 15점
- 방어자: 보병 1,000명, 전략 점수 15점
- 지형: 평야

**계산**:
```
공격자 능력치 = 1,000 × 1.0 × 1.2 (평야 공격) = 1,200
방어자 능력치 = 1,000 × 1.2 × 0.9 (평야 방어) = 1,080

공격자 전투력 = 1,200 × 1.15 (전략 15점) = 1,380
방어자 전투력 = 1,080 × 1.15 (전략 15점) = 1,242

승률 = 1,380 / (1,380 + 1,242) = 52.6%
```

**결과**: 공격자 약간 유리 (평야 지형 공격 보너스)

### C.2.2. 상성 적용 전투 시뮬레이션

**조건**:
- 공격자: 기병 500명, 전략 점수 20점
- 방어자: 보병 500명, 전략 점수 15점
- 지형: 초원

**계산**:
```
공격자 능력치 = 500 × 1.5 × 1.25 (초원) × 1.2 (상성) = 1,125
방어자 능력치 = 500 × 1.2 × 0.95 (초원) × 0.8 (상성) = 456

공격자 전투력 = 1,125 × 1.2 (전략 20점) = 1,350
방어자 전투력 = 456 × 1.15 (전략 15점) = 524.4

승률 = 1,350 / (1,350 + 524.4) = 72.0%
```

**결과**: 공격자 압도적 유리 (상성 + 지형 + 전략)

### C.2.3. 전략 점수 영향 분석

### C.2.4. 전투 밸런스 결론

## F.1. 시나리오 1: 평야에서의 정면 충돌

### F.1.1. 상황 설정

**공격자 (한국 - 서울)**:
- 병력: 보병 800명, 기병 200명
- 태세: 정면 돌진 (Charge)
- 전략 텍스트: "보병을 전면에 배치하고 기병으로 측면을 타격한다."
- 건물: 훈련소, 무기고

**방어자 (일본 - 도쿄)**:
- 병력: 보병 600명, 궁병 400명
- 태세: 견고한 방어 (Solid Defense)
- 전략 텍스트: "궁병을 후방에 배치하여 원거리 공격을 집중하고, 보병으로 전선을 유지한다."
- 건물: 성벽

**지형**: 평야

### F.1.2. 전투력 계산

**공격자 능력치**:
```
보병: 800 × 1.0 × 1.2 (평야 공격) = 960
기병: 200 × 1.5 × 1.2 (평야 공격) = 360
건물 보너스: 훈련소 +10%, 무기고 +20% = +30%
총 능력치: (960 + 360) × 1.3 = 1,716
```

**방어자 능력치**:
```
보병: 600 × 1.2 × 0.9 (평야 방어) = 648
궁병: 400 × 0.7 × 0.9 (평야 방어) = 252
건물 보너스: 성벽 +25% = +25%
총 능력치: (648 + 252) × 1.25 = 1,125
```

### F.1.3. LLM 전략 점수 판정

**공격자 전략 분석**:
- 지형 일치성: 7/10 (평야에서 기병 활용 적절)
- 병과 연계성: 4/5 (보병-기병 연계 적절)
- 병법 논리성: 6/10 (기본적인 측면 공격)
- **총점: 17/30**

**방어자 전략 분석**:
- 지형 일치성: 5/10 (평야에서 방어 불리)
- 병과 연계성: 5/5 (보병-궁병 연계 우수)
- 병법 논리성: 7/10 (원거리 공격 집중 적절)
- 첩보 카운터: 0/5 (첩보 정보 없음)
- **총점: 17/30**

### F.1.4. 최종 전투력 및 결과

**공격자 최종 전투력**:
```
1,716 × (1 + 17/100) = 1,716 × 1.17 = 2,007.72
```

**방어자 최종 전투력**:
```
1,125 × (1 + 17/100) = 1,125 × 1.17 = 1,316.25
```

**승률 계산**:
```
공격자 승률 = 2,007.72 / (2,007.72 + 1,316.25) = 60.4%
```

**결과**: 공격자 승리 (대승)

**병력 손실**:
- 공격자: 25% 손실 → 250명 손실
- 방어자: 65% 손실 → 650명 손실

```
   - 우호도 변경
6. 뉴스 생성
   - 주요 이벤트 뉴스 생성
   - 뉴스 피드 업데이트
7. 상태 업데이트
   - 도시 상태 업데이트
   - 외교 관계 업데이트
   - 점수 계산
8. 다음 턴 전환
```

## G.3. 전투 플로우

### G.3.1. 전투 시작

```
1. 공격자가 공격 명령 입력
2. 방어자에게 공격 알림 전송
3. 전투 화면 표시 (양측)
4. 전략 입력 시간 시작
```

### G.3.2. 전략 입력

```
1. 태세 선택 (Stance)
   - 공격 태세: 정면 돌진/포위/기습
   - 방어 태세: 견고한 방어/유연한 대응/반격
2. 세부 작전 입력 (100자 이내)
3. 병력 배치 확인
4. "전략 확정" 버튼 클릭
5. 상대방 입력 대기
```

### G.3.3. 전투 판정

```
1. 양측 전략 수집
2. 전투 컨텍스트 구성
   - 지형 정보
   - 병력 정보
   - 건물 정보
   - 첩보 정보
3. LLM API 호출
4. 전략 점수 산출
   - 지형 일치성 (10점)
   - 병과 연계성 (5점)
   - 병법 논리성 (10점)
   - 첩보 카운터 (5점, 방어자)
5. 능력치 점수 계산
6. 최종 전투력 계산
7. 승패 판정
8. 병력 손실 계산
```

### G.3.4. 전투 결과

```
1. 전투 결과 화면 표시
   - 승패 표시
   - 전략 점수 표시
   - 병력 손실 표시
   - LLM 판정 요약 표시
2. 타일 점령 처리 (승자)
3. 약탈 처리 (승자)
4. 뉴스 생성
5. 메인 화면으로 복귀
```
